Content-Focused Layout

Best for documentation, long-form articles, and research summaries.

Use case: Documentation Pattern: Content-focused

Technical guide

Layout decisions as information architecture.

Content-focused layouts prioritize a primary reading column with supporting sidebars. This model works for documentation because readers need depth, not distraction. Keep the main column between 600–700px for readability, and use subheads, callouts, and diagrams to reset the scan.

Eye-tracking studies show readers spend 80% of their time on the left half of the page. The opening paragraphs should deliver the summary, while the sidebar provides quick reference items like a table of contents, glossary, or download links.

Container-first layout strategy

Container queries let documentation modules adapt based on their available space instead of the viewport. This is critical when the same callout appears in a full article or inside a sidebar. Define a container on the module wrapper, then scale typography and layout inside.

@container (min-width: 400px) {
  .callout {
    display: grid;
    grid-template-columns: 1fr 2fr;
    gap: 1rem;
  }
}

Reflow, focus order, and scan rhythm

WCAG reflow requires content to work at 320px wide. Content-focused pages should collapse sidebars beneath the main column and avoid visual reordering that breaks keyboard focus. Keep headings short, use consistent spacing between paragraphs, and avoid justified text to prevent rivers.

For long pages, use content-visibility to skip off-screen rendering while preserving search and focus behavior. Reserve space with aspect ratios so images do not shift the layout as they load.

Reading width

50–75 characters per line for comfort.

CLS target

Reserve media space to stay at ≤ 0.1.
