Grid/Box Layout

Best for portfolios and equal-weight content libraries.

Use case: Portfolio Pattern: Grid/Box

Layout intent

Every project gets equal weight in a predictable grid.

Grid/box layouts keep portfolios scannable, with each tile carrying the same visual priority. They are ideal when the goal is to compare work quickly rather than tell a single story. The grid uses the RAM pattern (repeat, auto-fit, minmax) so tiles expand or wrap without breaking alignment.

RAM pattern

Repeat, auto-fit, minmax keeps the grid flexible while preserving tile widths. Use auto-fit when you want tiles to expand into empty space, and auto-fill when you need consistent column count.

.portfolio-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1.5rem;
}

Why it works

Comparison ready

Equal tile weight makes differences easy to spot.

Reusable modules

Cards can be lifted into other layouts with container queries.

Stable load

Aspect ratios prevent layout shift and keep CLS ≤ 0.1.
